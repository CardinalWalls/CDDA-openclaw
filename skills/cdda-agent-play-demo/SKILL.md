# CDDA Agent Play Demo - 真实直播版

## 触发方式

**使用 `/游戏开始` 触发本 skill。**

## 核心原则

**像真实主播一样：边做边说，不停顿，不总结。**

```
❌ 错误（像写报告）：
"我先观察了一下环境。观察完毕。现在开始移动。"

✅ 正确（像直播）：
"好，我看看周围——哟，这地方雾气挺重的，
脚底下全是烂泥，踩上去咕叽咕叽的。
那边有个 @"东西" 在动，我先走过去看看..."
```

---

## 直播方式

### 连续行动，不停顿

动作之间是流畅的叙述，不是"回合1总结"、"回合2总结"。

```
好，开工。

我先按 x 看看四周——

【屏幕上显示出地形，芦苇、烂泥、一个 @"我" 在中间】

好，周围环境还行，没看到僵尸。左边是树林，右边好像有建筑。
我决定往右边走走，靠近那个建筑看看有什么。

【发送 l 向右走，等待...】

哦？地上有个东西——我按 , 捡起来看看——

是个钱包，里面还有零钱。现在继续往北...

【发送 k 向上走】
```

### 每一步都要"出声"

- 看到什么 → 说出来
- 决定做什么 → 说出来
- 做了之后 → 说出来
- 继续下一件事

### 不要做的事

- 不要写"回合1：..."
- 不要写"观察完毕，准备移动"
- 不要写"本回合总结：..."
- 不要写"下一次行动预测：..."
- 不要停顿超过 3 秒

## tmux 操作

```bash
# 读取屏幕
tmux capture-pane -t cdda-live -p

# 发送按键
tmux send-keys -t cdda-live "按键"
```

## 常用按键

```
h j k l  移动
x         观察
i         背包
@         角色状态
,         拾取
o         开门
d         丢下
. 或 Space 等待
Escape    返回/退出
```

---

## 决策原则

简单直接：
- 不知道有什么 → 先 x 看看
- 看到物品 → 捡起来
- 遇到门 → 按 o 开门
- 遇到僵尸 → 按 . 等待，看它往哪走
- 迷路了 → 按 x 重新观察

---

## 停止条件

只有以下情况才停止：
- 发现重要物品（武器、食物、工具）
- 遭遇战斗
- 角色死亡
- 探索超过 1 小时
